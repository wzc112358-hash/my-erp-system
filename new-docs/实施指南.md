#### 步骤 1: PocketBase 后台添加 manager_confirmed 字段

**类型**: 手动操作 (PocketBase 后台)  
**依赖**: 无  
**预计时间**: 15 分钟

**操作说明**:

登录 PocketBase 管理后台，为以下 5 个集合添加 `manager_confirmed` 字段：

| 集合 | 字段名 | 类型 | 可选值 |
|------|--------|------|--------|
| sale_invoices | manager_confirmed | select | pending,approved,rejected |
| purchase_invoices | manager_confirmed | select | pending,approved,rejected |
| sale_receipts | manager_confirmed | select | pending,approved,rejected |
| purchase_payments | manager_confirmed | select | pending,approved,rejected |
| purchase_arrivals | manager_confirmed | select | pending,approved,rejected |

**注意**: 默认值 `pending` 在后端 hooks 中设置，不在 PocketBase 后台配置。

**操作步骤**:

1. 登录 PocketBase 管理后台 (api.henghuacheng.cn)
2. 进入「Collections」管理页面
3. 选择 `sale_invoices` 集合
4. 点击「New field」，选择「Select」类型
5. 字段名称输入 `manager_confirmed`
6. 在 Options 中添加三个值: `pending`, `approved`, `rejected`
7. 保存字段
8. 重复步骤 3-7 为其他 4 个集合添加相同字段

**验证方法**:

- 检查 5 个集合是否都已添加 `manager_confirmed` 字段
- 检查字段类型是否为 Select
- 检查可选值是否为 `pending`, `approved`, `rejected`

#### 步骤 2: PocketBase 后台配置字段权限

**类型**: 手动操作 (PocketBase 后台)  
**依赖**: 步骤 1 完成  
**预计时间**: 10 分钟

**操作说明**:

配置 `manager_confirmed` 字段的访问权限：

| 角色 | 权限 |
|------|------|
| 销售 (sales) | 只读 (View) |
| 采购 (purchasing) | 只读 (View) |
| 经理 (manager) | 读写 (Create, View, Update) |

**操作步骤**:

1. 在 PocketBase 后台，选择 `sale_invoices` 集合
2. 进入「Rules」或「Permissions」配置页面
3. 为 `manager_confirmed` 字段配置权限:
   - sales 角色: 只勾选「View」
   - purchasing 角色: 只勾选「View」
   - manager 角色: 勾选「Create」、「View」、「Update」
4. 保存权限配置
5. 重复步骤 1-4 为其他 4 个集合配置相同权限

**验证方法**:

- 使用销售账号登录，确认无法修改 `manager_confirmed` 字段
- 使用采购账号登录，确认无法修改 `manager_confirmed` 字段
- 使用经理账号登录，确认可以修改 `manager_confirmed` 字段

#### 步骤 3: 修改后端 Hooks 设置默认值和添加通知逻辑

**类型**: 代码修改  
**依赖**: 步骤 1-2 完成

**修改文件列表**:

| 文件 | 修改内容 |

| `backend/hooks/sale_receipt.go` | 创建时设置默认值为 pending，变更时创建通知 |
| `backend/hooks/purchase_payment.go` | 创建时设置默认值为 pending，变更时创建通知 |
| `backend/hooks/purchase_arrival.go` | 创建时设置默认值为 pending，变更时创建通知 |

**3.1 设置默认值 (OnRecordCreate hook)**

由于 PocketBase 后台无法设置 select 字段的默认值，需要在 hooks 中手动设置。新建记录时，默认将 `manager_confirmed` 设置为 `pending`。

在每个 hook 文件的 `OnRecordCreate` 中添加:

```go
// 示例: sale_invoice.go 中的修改
app.OnRecordCreate("sale_invoices").Bind(&hook.Handler[*core.RecordEvent]{
    Func: func(e *core.RecordEvent) error {
        record := e.Record()
        // 设置默认值为 pending
        record.Set("manager_confirmed", "pending")
        return e.Next()
    },
    Priority: 0,
})
```

**3.2 通知存储规则**:

| 确认操作类型 | 通知存储集合 | 接收人 |
|-------------|-------------|--------|
| 销售开票确认 (sale_invoices) | `notifications_02` | 创建该记录的销售用户 |
| 采购收票确认 (purchase_invoices) | `notifications` | 创建该记录的采购用户 |
| 销售收款确认 (sale_receipts) | `notifications_02` | 创建该记录的销售用户 |
| 采购付款确认 (purchase_payments) | `notifications` | 创建该记录的采购用户 |
| 采购发货确认 (purchase_arrivals) | `notifications` | 创建该记录的采购用户 |

**3.3 实现通知逻辑**:

在需要修改的 hook 文件的 `OnRecordUpdate` 中添加:

```go
// 示例: sale_invoice.go 中的修改
app.OnRecordUpdate("sale_invoices").Bind(&hook.Handler[*core.RecordEvent]{
    Func: func(e *core.RecordEvent) error {
        oldRecord, _ := e.OldRecord()
        newRecord := e.Record()
        
        oldStatus := oldRecord.GetString("manager_confirmed")
        newStatus := newRecord.GetString("manager_confirmed")
        
        // 仅当 pending -> approved 时发送通知
        if oldStatus == "pending" && newStatus == "approved" {
            creatorId := newRecord.GetString("creator")
            invoiceNo := newRecord.GetString("no")
            
            title := "销售开票已确认"
            message := fmt.Sprintf("经理已确认您的销售开票申请，发票号：%s", invoiceNo)
            
            // 销售的通知存储到 notifications_02
            CreateNotification02(app, "manager_confirm", title, message, creatorId, "")
        }
        
        return e.Next()
    },
    Priority: 0,
})
```

**各文件具体通知内容**:

| 文件 | 通知标题 | 通知内容模板 | 存储集合 |
|------|----------|-------------|----------|
| sale_invoice.go | 销售开票已确认 | 经理已确认您的销售开票申请，发票号：{no} | notifications_02 |
| purchase_invoice.go | 采购收票已确认 | 经理已确认您的采购收票申请，发票号：{no} | notifications |
| sale_receipt.go | 销售收款已确认 | 经理已确认您的销售收款登记，收款金额：{amount} | notifications_02 |
| purchase_payment.go | 采购付款已确认 | 经理已确认您的采购付款申请，付款金额：{amount} | notifications |
| purchase_arrival.go | 采购发货已确认 | 经理已确认您的采购发货信息，发货批次：{tracking_contract_no} | notifications |

**验证方法**:

1. 在销售页面创建发票，使用经理账号确认该发票
2. 检查 `notifications_02` 集合中是否有新通知
3. 确认通知接收人是发票创建者
4. 重复以上测试对采购相关记录进行验证

**修改内容**:

```typescript
import PocketBase from 'pocketbase';

const getCurrentSystem = (): string => {
  return localStorage.getItem('erp_system') || 'beijing';
};

const getApiBaseUrl = (): string => {
  const system = getCurrentSystem();
  switch (system) {
    case 'beijing':
      return 'https://api-beijing.henghuacheng.cn';
    case 'lanzhou':
      return 'https://api-lanzhou.henghuacheng.cn';
    default:
      return 'https://api-beijing.henghuacheng.cn';
  }
};

export const pb = new PocketBase(getApiBaseUrl());
export const API_BASE_URL = getApiBaseUrl();
export const getCurrentSystemName = (): string => {
  const system = getCurrentSystem();
  return system === 'beijing' ? '北京' : '兰州';
};
```

**⚠️ 本地开发注意事项**:

> **部署阶段**：完成全部实施修改后，需将 `http://127.0.0.1:8090` 替换回生产域名：
> - 北京系统: `https://api-beijing.henghuacheng.cn`
> - 兰州系统: `https://api-lanzhou.henghuacheng.cn`

```typescript
getContractDetail: async (salesContractId: string) => {
  // 1. 获取销售合同 (expand customer)

  // 3. 并行获取 6 个子表的原始记录列表:
  //    - sales_shipments  (filter: sales_contract=salesContractId)
  //    - sale_invoices    (filter: sales_contract=salesContractId)

  //    - purchase_invoices  (filter: purchase_contract IN purchaseContractIds)
  //    - purchase_payments  (filter: purchase_contract IN purchaseContractIds)
  // 4. 复用 getComparisonData 的利润计算逻辑
  // 5. 返回 ContractDetailData 结构
}
```

```typescript
interface ContractDetailData {
```

**10.6 利润分析公式**:

> **注意**: 现有 `ComparisonPage.tsx` 中的营业利润公式使用的是不含税价（`销售含税/1.13 - 采购含税/1.13`），与本文档不一致。ContractDetailPage 应以本文档公式为准。

| 指标 | 公式 |
|------|------|
| 销售总金额（含税） | sales_contracts.total_amount |
| 采购总金额（含税） | SUM(purchase_contracts.total_amount) (所有关联采购合同) |
| 销售总金额（不含税） | total_amount / 1.13 |
| 采购总金额（不含税） | total_amount / 1.13 |
| 运费合计 | SUM(purchase_arrivals.freight_1 + purchase_arrivals.freight_2) |
| 杂费合计 | SUM(purchase_arrivals.miscellaneous_expenses) |
| 营业利润 | 销售含税 - 采购含税 - 运费 - 杂费 |
| 税额 | (销售含税 - 采购含税) * 0.1881 |
| 净利润 | 销售含税 - 采购含税 - 税额 - 运费 - 杂费 |
| 数量匹配检查 | sales_contracts.total_quantity == SUM(purchase_contracts.total_quantity) |

> **净利润 = 营业利润 - 税额**，展开即：`销售含税 - 采购含税 - 运费 - 杂费 - (销售含税 - 采购含税) * 0.1881`
